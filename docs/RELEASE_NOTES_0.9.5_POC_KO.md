# Samsung Switch Watch 0.9.5-poc 릴리스 노트

## 핵심 변경

이번 버전은 Viewer의 주기 감시와 로컬 설정 저장 실패를 정확히 구분하고,
인증 실패 시 장비 계정 잠금 위험을 줄이는 안정성 릴리스입니다.

- 감시 주기의 예상하지 못한 내부 오류를 감시 이력 저장 오류로 잘못 안내하지 않습니다.
- 감시 이력의 I/O·권한 오류는 기존 `VIEWER_MONITOR_STATE_WRITE_FAILED`로 유지합니다.
- 그 밖의 감시 주기 오류는 `VIEWER_MONITOR_CYCLE_FAILED`로 구분하며 다음 주기에 다시 시도합니다.
- Viewer 설정 저장 실패는 `VIEWER_SETTINGS_WRITE_FAILED`로 표시하고 Agent 연결 장애와 구분합니다.
- 초기 연결과 새로고침 중 설정 저장만 실패해도 정상 Agent 연결 상태를 유지합니다.
- 연결 설정을 디스크에 저장하지 못하면 기존 Agent 연결과 설정을 그대로 유지합니다.
- 창 위치, 이벤트 커서와 종료 시 설정 저장 실패도 조용히 무시하지 않고 안전한 진단 코드를 남깁니다.
- 인증 실패는 감시 이력이나 장비 설정 파일 저장보다 먼저 런타임에서 차단합니다.
  두 저장 작업이 함께 실패해도 같은 실행에서 잘못된 비밀번호로 다시 접속하지 않습니다.

## 진단 정보와 보안

- Viewer 진단 로그에는 시각, 허용된 단계와 안정 오류 코드만 기록합니다.
- 예외 원문, 파일 경로, Agent 주소, 장비 IP, 계정, 비밀번호와 명령 출력은 기록하지 않습니다.
- 설정 파일의 원자적 교체와 기존 파일 보존 방식은 변경하지 않았습니다.
- Agent API, Viewer 설정 JSON, 감시 상태 JSON과 장비 등록 데이터 형식은 변경하지 않았습니다.

## 호환성

- Agent와 Viewer의 역할, 설치·업데이트 흐름과 HTTPS/18443 통신은 변경하지 않았습니다.
- 장비에 전달되는 읽기 전용 `show` 명령, Telnet 세션 제한과 모델별 호환 명령 탐색은 변경하지 않았습니다.
- UI 배치와 운영자의 기본 작업 순서는 변경하지 않았습니다.

## 검증

- 감시 이력 `IOException`과 예상하지 못한 예외의 오류 코드 분리
- 오류 안내 UI 전달이 실패해도 감시 루프가 종료되지 않는 경로
- 초기 연결 중 Viewer 설정 저장 실패 시 Agent 연결 상태 유지
- 지연된 이벤트 커서 저장 실패의 사용자 안내와 진단 기록
- 연결 설정 저장 실패 시 기존 클라이언트와 설정 유지
- 감시 이력과 장비 설정 저장이 동시에 실패한 인증 오류의 즉시 차단
- 신규 진단 단계·코드 허용과 민감정보 미기록
- Core, Agent, Viewer 전체 자동화 테스트
- Release 빌드, 패키지 계약, PowerShell 5.1 구문, Git 공백과 GitHub 릴리스 계약 검증

## 알려진 제한

- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어에서 검증하기 전까지는 POC 상태입니다.
- 실제 사내 방화벽, EDR, Windows 서비스 계정과 장시간 운영은 사내 단계에서 확인해야 합니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없으며 사내 EDR 또는 WDAC 승인이 필요할 수 있습니다.
