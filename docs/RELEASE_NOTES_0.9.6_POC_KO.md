# Samsung Switch Watch 0.9.6-poc 릴리스 노트

## 핵심 변경

이번 버전은 Viewer 설정을 여러 작업이 동시에 저장할 때 최신 값이 이전 값으로
되돌아갈 수 있는 경쟁 조건을 제거한 안정성 릴리스입니다.

- 창 위치 저장, 이벤트 커서 저장, Agent 연결 변경과 종료 저장을 하나의 설정 저장 경계로
  직렬화합니다.
- 설정 파일을 기록하기 전에 Dictionary를 포함한 일관된 스냅샷을 만들므로 인증서 신뢰 정보나
  이벤트 커서를 열거하는 중 변경되어 저장이 실패하지 않습니다.
- 연결 설정 창이 열린 동안 이벤트 커서나 창 위치가 바뀌어도 연결 적용 시 최신 런타임 값을
  병합해 보존합니다.
- 로그 보존 기간 초기화로 최신 이벤트 커서가 이전보다 낮아진 경우에도 오래된 연결 창의 높은
  커서를 되살리지 않으며, 연결 검증 첫 요청부터 낮아진 최신 커서를 사용합니다.
- 처음 연결한 Agent의 기준 커서는 누락 이벤트가 있었던 것으로 오판하지 않아 불필요한 동기화
  알림을 만들지 않습니다.
- 새 Agent 연결의 주소, 자동 시작 선택과 대상 Agent 신뢰 정보는 연결 창의 확정 값을 사용합니다.
- 이전 백그라운드 저장은 연결 변경이 완료된 뒤 새 설정을 덮어쓰지 못합니다.
- 외부 코드가 `CurrentSettings` 결과를 수정해 Viewer 내부 설정을 우회 변경하지 못하도록
  방어 복사본을 반환합니다.

## 데이터와 호환성

- `viewer-settings.json`의 경로, JSON 필드명과 값 형식은 변경하지 않았습니다.
- 개별 파일 기록은 기존과 같이 같은 폴더의 임시 파일을 완성한 뒤 원자적으로 교체합니다.
- 설정 저장 실패 시 기존 오류 코드 `VIEWER_SETTINGS_WRITE_FAILED`와 연결 변경의 fail-closed
  동작을 유지합니다.
- Agent API, 장비 등록·자격 증명 형식, 감시 상태 JSON과 이벤트 형식은 변경하지 않았습니다.

## 운영 흐름

- Agent와 Viewer의 역할, 설치·업데이트 흐름과 HTTPS/18443 통신은 변경하지 않았습니다.
- 장비에 전달되는 읽기 전용 `show` 명령, Telnet 세션 제한과 모델별 호환 명령 탐색은
  변경하지 않았습니다.
- UI 배치와 운영자의 기본 작업 순서는 변경하지 않았습니다.

## 검증

- 동시 설정 writer가 한 번에 하나만 파일 저장소에 진입하는지 확인
- 먼저 시작한 저장이 지연된 상태에서도 나중 스냅샷이 최종 파일에 남는지 확인
- 첫 저장이 I/O 오류로 실패해도 다음 저장이 교착 없이 완료되는지 확인
- 인증서 신뢰 정보 변경 중 설정 스냅샷이 일관되게 생성되는지 확인
- 오래된 연결 창 후보를 적용해도 최신 창 위치·이벤트 커서·다른 Agent 신뢰 정보가 유지되는지 확인
- 이벤트 커서 `500 → 120` 초기화와 32개 커서 제한에서 최신 값·키가 보존되는지 확인
- 신규 Agent 기준선 이후 실제 변경은 적용되지만 잘못된 누락 이벤트 요약은 발생하지 않는지 확인
- 연결 후보의 주소·자동 시작 선택·대상 Agent 신뢰 정보가 최종 설정에 반영되는지 확인
- `CurrentSettings` 방어 복사와 기존 연결 저장 실패의 fail-closed 회귀 확인
- Core, Agent, Viewer 전체 자동화 테스트
- Release 빌드, 패키지 계약, PowerShell 5.1 구문, Git 공백과 GitHub 릴리스 계약 검증

## 알려진 제한

- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어에서 검증하기 전까지는 POC 상태입니다.
- 실제 사내 방화벽, EDR, Windows 서비스 계정과 장시간 운영은 사내 단계에서 확인해야 합니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없으며 사내 EDR 또는 WDAC 승인이 필요할 수 있습니다.
