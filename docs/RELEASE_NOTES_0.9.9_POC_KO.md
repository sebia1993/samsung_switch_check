# Samsung Switch Watch 0.9.9-poc 릴리스 노트

## 핵심 변경

이번 버전은 릴리스 준비 과정에서 버전 표기 일부가 누락되는 문제를 자동으로
차단하는 배포 안정성 개선 릴리스입니다.

- GitHub 릴리스 워크플로의 기본 버전을 단일 기준으로 사용합니다.
- Windows CI, 배포 스크립트, 설치 안내, 저장소 안내와 사용설명서 생성기의
  활성 버전이 기준과 일치하는지 자동 검사합니다.
- Agent 및 Viewer ZIP 파일명, CI 아티팩트 이름과 패키지 계약의 버전도 함께
  검증합니다.
- README의 현재 버전, 배포 파일명, 빌드 명령과 현재 릴리스 노트 링크를
  각각 확인합니다.
- 의도적으로 오래된 버전 문자열을 검사에 넣어 누락 감지 규칙 자체가 실제로
  실패하는지 자체 검증합니다.
- 별도 배포 도우미 검사는 특정 버전을 중복 하드코딩하지 않고 유효한
  시맨틱 버전 형식인지 확인합니다.

## 호환성

- Agent API, Viewer 화면, HTTPS 연결, Telnet 장비 접속과 저장 형식은 변경하지
  않았습니다.
- 장비에 전달되는 명령, 접속 주기, 재시도와 세션 제한은 변경하지 않았습니다.
- 기존 `0.9.8-poc` 설정과 데이터는 그대로 사용할 수 있습니다.
- `CurrentUserSecretProtector`의 `v0.8` 문자열은 기존 암호화 데이터 복호화
  호환성 값이므로 변경하지 않았습니다.
- 과거 릴리스 노트와 기존 Git 태그는 역사 기록으로 그대로 보존합니다.

## 검증

- PowerShell 5.1 구문 분석과 배포 도우미 계약 테스트
- 활성 버전 일관성 계약 및 오래된 버전 자체 거부 테스트
- Core, Agent, Viewer 전체 자동화 테스트
- Mock Agent EXE 통신 테스트
- DOCX/PDF 사용설명서 전 페이지 렌더링 및 시각 검토
- ZIP 내부 파일, Product Version, 소스 커밋과 해시 계약 테스트
- Windows CI 패키지 재다운로드 및 해시 검증

## 알려진 제한

- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어에서 검증하기 전까지는 POC 상태입니다.
- 실제 사내 EDR, 방화벽, 사용자 권한과 장시간 운영은 사내 단계에서 확인해야 합니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없으며 사내 EDR 또는 WDAC 승인이 필요할 수
  있습니다.
