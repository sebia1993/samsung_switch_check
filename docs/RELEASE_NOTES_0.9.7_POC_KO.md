# Samsung Switch Watch 0.9.7-poc 릴리스 노트

## 핵심 변경

이번 버전은 장비 관리 중 로컬 설정 파일을 읽거나 저장하지 못했을 때 Viewer 전체가
종료될 수 있던 경로를 막은 안정성 릴리스입니다.

- 장비 선택 중 계정 파일 읽기가 실패해도 장비 관리 창과 Viewer를 계속 유지합니다.
- 기존 장비 저장 전 계정 재확인과 실제 파일 저장을 하나의 오류 경계 안에서 처리합니다.
- 장비 삭제 파일 갱신이 실패하거나 대상이 사라진 경우 삭제된 것처럼 화면을 초기화하지
  않습니다.
- 실패 전 입력값과 선택 장비를 유지하므로 권한·잠금 문제를 해결한 뒤 다시 시도할 수
  있습니다.
- 장비 관리 창을 닫으며 대시보드를 갱신하는 중 발생한 예외도 앱 전체 종료로 전파하지
  않습니다.
- 장비 목록 읽기 실패를 `등록 장비 0대`로 오인하지 않고 기존 목록·선택·입력값을
  유지합니다.
- 장비 설정 파일 저장이 완료된 뒤 감시 이력 정리만 실패한 경우 전체 저장 실패가 아닌
  부분 성공으로 표시합니다.
- 파일 읽기, 파일 저장, 장비 없음과 예상 밖 오류를 안정 코드로 구분합니다.
- 임의 예외 본문은 검증 문구로 표시하지 않고, 미리 정한 안전 문구만 사용자 화면에
  표시합니다.
- 진단 로그에는 단계와 안정 코드만 기록하며 장비 IP, 계정, 비밀번호, 파일 경로와 예외
  본문을 기록하지 않습니다.

## 데이터와 호환성

- `viewer-devices.json`의 경로, JSON 스키마, 암호화 값과 저장 형식은 변경하지 않았습니다.
- Agent API, 감시 상태, 이벤트 형식과 Viewer 설정 파일은 변경하지 않았습니다.
- 정상 저장·삭제 흐름과 접속 시험 전 주기 감시 차단 정책은 그대로 유지합니다.
- 새 진단 코드 `VIEWER_DEVICE_STORE_WRITE_FAILED`는 장비 설정 파일을 기록하지 못한 경우에만
  표시합니다.
- `VIEWER_MONITOR_STATE_WRITE_FAILED`는 장비 설정 저장은 완료됐지만 감시 이력 정리가
  실패한 경우에도 사용하며, 이때 장비를 중복 등록하거나 다시 저장할 필요는 없습니다.

## 운영 흐름

- 장비 IP, 모델, ID, 로그인 PW와 선택적 enable PW를 Viewer에 입력하는 기존 순서는
  변경하지 않았습니다.
- Agent와 Viewer의 역할, HTTPS/18443 통신과 Telnet 세션 동작은 변경하지 않았습니다.
- 장비에 전달되는 읽기 전용 `show` 명령과 접속 주기는 변경하지 않았습니다.

## 검증

- 장비 선택 중 설정 파일 읽기 오류를 주입해 선택과 입력값이 유지되는지 확인
- 저장 중 파일 잠금·권한 오류를 주입해 창이 유지되고 재시도 가능한지 확인
- 삭제 중 원자 파일 저장 오류를 주입해 장비가 삭제된 것처럼 표시되지 않는지 확인
- 오류 해제 후 같은 창에서 정상 삭제가 가능한지 회귀 확인
- 사용자 메시지와 진단 기록에 합성 IP, 계정과 예외 본문이 포함되지 않는지 확인
- 최초 열기와 새로고침 중 읽기 실패가 빈 장비 목록으로 표시되지 않는지 확인
- 감시 이력 파일 쓰기 오류를 주입해 장비 설정은 저장되고 부분 성공 경고가 표시되는지 확인
- 임의 `InvalidDataException` 본문이 화면·대시보드·진단에 노출되지 않는지 확인
- 예외 종류별 안정 코드 매핑과 진단 allowlist 확인
- Core, Agent, Viewer 전체 자동화 테스트
- Release 빌드, 패키지 계약, PowerShell 5.1 구문, Git 공백과 GitHub 릴리스 계약 검증

## 알려진 제한

- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어에서 검증하기 전까지는 POC 상태입니다.
- 실제 사내 파일 잠금, EDR, 사용자 프로필 권한과 장시간 운영은 사내 단계에서 확인해야
  합니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없으며 사내 EDR 또는 WDAC 승인이 필요할 수 있습니다.
