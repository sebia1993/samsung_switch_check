# IES4224GP 현장 POC 체크리스트

실제 회사망 자료를 외부로 반출하지 않고 현장에서 판정할 수 있는 체크리스트입니다.

## 설치 전

- [ ] Agent PC에서 대상 스위치 TCP/23 연결 가능
- [ ] Viewer PC에서 Agent PC TCP/18443 연결 가능
- [ ] 원격 PC에 서비스와 방화벽 규칙을 설치할 관리자 권한이 있음
- [ ] 스위치 조회 전용 Telnet 계정 준비
- [ ] Agent IP만 허용하는 관리 ACL 검토
- [ ] PC 시간 동기화 정상

## 최초 프로브

- [ ] `show version` 성공 또는 `PARSER_UNSUPPORTED`로 안전 분리
- [ ] `show system`에서 가동 시간(uptime) 추출
- [ ] `show log ram` 최초 결과는 알림 없이 기준선 처리
- [ ] `show interfaces status`에서 포트 상태 추출
- [ ] 지원되지 않는 명령 때문에 장비 전체가 장애로 표시되지 않음
- [ ] 로그에 사용자명·비밀번호·실제 원문이 노출되지 않음

## Agent 지속 동작

- [ ] RDP 로그오프 후에도 수집 지속
- [ ] Agent PC 재부팅 후 서비스 자동 시작
- [ ] 한 장비당 동시 Telnet 세션이 하나 이하
- [ ] 시간 초과(timeout) 후 세션이 정리되고 다음 주기에 재시도
- [ ] SQLite 파일이 로컬 디스크에 있으며 네트워크 공유에 놓이지 않음

## Viewer

- [ ] 최초 페어링 후 인증서 지문 확인
- [ ] 전체 대시보드가 1280x720 이상에서 읽힘
- [ ] 미니 창 이동·항상 위·위치 기억 동작
- [ ] 시스템 트레이로 숨기기와 명시적 종료가 구분됨
- [ ] Viewer가 종료된 동안 발생한 이벤트가 재실행 후 다시 동기화됨
- [ ] Agent 연결이 끊겨도 모든 스위치를 DOWN으로 표시하지 않음
- [ ] Telnet 수집 실패가 실제 주소 없이 표준화 오류 코드로 표시되고 반복 알림되지 않음
- [ ] 다음 정상 수집에서 수집기 복구 이벤트가 한 번 발생함

## 이벤트 시나리오

- [ ] 최초 기준선에서 기존 로그 폭탄이 발생하지 않음
- [ ] 신규 로그가 한 번만 알림됨
- [ ] 중요 포트 `UP -> DOWN` 신규 장애 발생
- [ ] 같은 DOWN 상태가 다음 주기에도 반복 팝업되지 않음
- [ ] 확인 버튼을 눌러도 장애가 복구 처리되지 않음
- [ ] `DOWN -> UP` 복구 이벤트와 지속 시간 생성
- [ ] 가동 시간(uptime) 감소가 재부팅으로 표시됨
- [ ] 로그 순환/초기화가 전체 신규 로그로 오인되지 않음

## 반출 가능한 진단 정보

- [ ] 표준화 오류 코드
- [ ] 단계 이름과 소요 시간
- [ ] 익명 장비 ID
- [ ] 비식별 모델·펌웨어 버전
- [ ] 원문(raw data) 없이 집계된 성공/실패 횟수

실제 IP, 호스트명, 계정명, MAC, 시리얼, 명령 원문, 인증서는 반출하지 않습니다.
