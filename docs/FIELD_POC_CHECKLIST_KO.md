# IES4224GP 현장 POC 체크리스트

실제 회사망 자료를 외부로 반출하지 않고 현장에서 판정하는 `v0.2.0-poc` 체크리스트입니다.

## 패키지와 설치

- [ ] GitHub Release의 Agent ZIP, Viewer ZIP, `SHA256SUMS.txt` 파일명과 버전 일치
- [ ] 두 ZIP의 SHA-256 일치
- [ ] ZIP 루트에서 `install-*.ps1 -Preflight` 성공, 시스템 변경 없음
- [ ] 문서의 기본 설치 명령으로 설치 성공
- [ ] Agent 서비스 이름이 `SamsungSwitchWatchAgent`로 고정됨
- [ ] 설치 실패를 유도했을 때 서비스·방화벽·프로그램 폴더가 이전 상태로 복구됨
- [ ] `-Repair` 후 DB·자격 증명·설정·인증서 보존
- [ ] Viewer 업데이트 실패 시 이전 실행 파일 복구, 화면 설정·토큰 보존

## 네트워크와 보안

- [ ] Agent PC에서 대상 스위치 IPv4 TCP/23 연결 가능
- [ ] Viewer PC에서 Agent PC TCP/18443 연결 가능
- [ ] 스위치 조회 전용 Telnet 계정과 Agent IP 제한 ACL 준비
- [ ] Agent 프로그램·데이터 ACL에 고정 서비스 SID만 포함
- [ ] 비밀번호·토큰·실제 IP·호스트명·MAC·명령 원문이 진단에 없음
- [ ] 코드 미서명과 Telnet 비암호화 제한을 운영자가 인지함

## Agent 지속 동작

- [ ] RDP 로그오프와 Agent PC 재부팅 후 수집 지속
- [ ] 동시에 도래한 명령이 한 Telnet 세션에서 수행됨
- [ ] 한 명령의 성공이 다른 명령의 실패를 해제하지 않음
- [ ] 일시적 실패 1~2회는 장애가 아니고 3회째에 장애 발생
- [ ] 인증 실패 1회 후 반복 로그인 중단, 자격 증명 검증 후 재개
- [ ] 명령·세션 제한시간 후 연결 정리와 다음 주기 재시도
- [ ] `/health/live`와 `/health/ready`가 프로세스와 준비 실패를 구분
- [ ] SQLite 손상 시 DB를 삭제하지 않고 readiness 503과 안정 오류 코드 표시

## 파서와 이벤트

- [ ] 최초 로그는 알림 없이 기준선 처리
- [ ] 불완전 출력이 마지막 정상값을 덮어쓰거나 업링크 Down을 만들지 않음
- [ ] 지원되지 않는 명령이 장비 전체 장애로 표시되지 않음
- [ ] 신규 로그와 중요 포트 `UP -> DOWN`이 한 번만 알림됨
- [ ] 확인이 복구로 처리되지 않고, `DOWN -> UP`에서 원래 장애가 복구됨
- [ ] 로그 순환·빈 버퍼·재부팅에서 신규 로그 폭주 없음
- [ ] 재부팅 Warning이 장비를 영구 Critical로 유지하지 않음

## Viewer 복원력

- [ ] Viewer 종료 중 1,201건 이상 변경 후 페이지 누락 없이 최종 상태 일치
- [ ] 재연결 중 ACK·복구·실시간 이벤트 순서가 섞여도 연속 커서 유지
- [ ] 장시간 Agent 중단 후 Viewer 재시작 없이 자동 복구
- [ ] catch-up 이벤트는 개별 팝업 대신 요약 알림
- [ ] 60초 스냅샷 갱신과 재연결 직후 즉시 갱신 동작
- [ ] Agent 연결 단절 시 장비 Down이 아니라 `현재 미확인` 표시
- [ ] 두 번째 Viewer 실행이 기존 창만 활성화
- [ ] 네트워크 작업 중 종료해도 5초 안에 프로세스 종료

## 반출 가능한 진단

- [ ] 표준화 오류 코드
- [ ] 단계 이름과 소요 시간
- [ ] 익명 장비 ID
- [ ] 비식별 모델·펌웨어 버전
- [ ] 원문 없이 집계된 성공·실패 횟수

실제 IP, 호스트명, 계정명, MAC, 시리얼, 명령 원문, 인증서와 토큰은 반출하지 않습니다.
