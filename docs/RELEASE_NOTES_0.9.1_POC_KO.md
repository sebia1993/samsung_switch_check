# Samsung Switch Watch 0.9.1-poc 릴리스 노트

## 핵심 변경

이번 버전은 Viewer 감시 이력의 활성 장애 보존 문제를 수정한 안정성 패치입니다.

- 활성 접속 장애와 활성 포트 장애가 이벤트 500건 보존 한도에 밀려 삭제되지 않습니다.
- 이벤트 정리는 각 저장 트랜잭션의 상태 변경이 끝난 뒤 한 번만 수행합니다.
- 500건을 초과하면 오래된 비활성 이력부터 삭제합니다.
- 활성 장애만 500건을 넘는 극단 상황에서는 데이터 무결성을 우선하여 일시적으로
  500건을 초과해 저장합니다.
- 활성 장애의 확인, 오류 코드 갱신과 복구 처리가 이벤트 정리 이후에도 계속 동작합니다.

## 호환성

- Agent API, Viewer 화면과 장비 명령은 변경하지 않았습니다.
- `viewer-monitor-state.json` 스키마는 기존 버전 3을 유지합니다.
- 기존 Viewer 장비 설정과 DPAPI 보호 계정은 그대로 사용합니다.
- Agent와 Viewer 설치·업데이트 방식은 `0.9.0-poc`과 같습니다.

## 검증

- 활성 접속·포트 장애 2건과 일반 이력 600건을 함께 저장하는 회귀 테스트
- 활성 포트 장애만 501건인 경계 상태의 저장·재시작·확인·복구 테스트
- Core, Agent와 Viewer 전체 자동화 테스트
- Release 빌드, 패키지 계약, PowerShell 5.1 구문과 GitHub 릴리스 계약 검사

## 알려진 제한

- Viewer 화면의 최근 이벤트 조회는 최신 500건으로 제한됩니다. 활성 장애 자체가
  500건을 넘는 비정상적인 상황에서는 저장 파일에는 보존되지만 오래된 일부 항목이
  화면 목록에 보이지 않을 수 있습니다.
- Viewer가 종료되어 있는 동안에는 주기 감시가 실행되지 않습니다.
- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어 검증 전에는 POC 상태입니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없는 경우 사내 EDR·WDAC 승인이 필요할 수 있습니다.
