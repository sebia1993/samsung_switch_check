# Samsung Switch Watch 0.9.4-poc 릴리스 노트

## 핵심 변경

이번 버전은 Viewer가 Agent의 신원을 안전하게 확인한 뒤 같은 감시 주기에서 그 결과를 재사용하도록 개선한 안정성 릴리스입니다.

- 감시 주기 시작의 명시적 연결 확인은 이전처럼 Agent 신원을 매번 다시 검증합니다.
- 신원 확인이 성공한 뒤 장비별 Telnet 조회와 호환 명령 탐색은 같은 검증 결과를 재사용합니다.
- 정상 감시 주기의 `/api/v4/identity` 호출 수를 `장비 수 + 1회`에서 주기당 1회로 줄였습니다.
- 여러 장비 조회가 동시에 처음 시작돼도 최초 신원 확인은 한 번만 수행합니다.
- Agent 전송 실패, 비정상 응답 또는 연결 상태가 오래된 경우 다음 조회 전에 신원을 다시 확인합니다.
- 사용자가 명시적으로 연결 확인을 다시 실행했는데 실패하면 이전 신원 캐시로 장비 조회를 계속하지 않습니다.
- Agent 인증서 공개키 변경이 한 번 감지되면 병렬 정상 요청이 해당 경고를 지우지 못하도록 동시성 처리를 보강했습니다.

## 보안 및 호환성

- HTTPS 인증서 공개키 확인, identity 응답 지문 확인, Viewer에 저장된 Agent 신뢰 정보 비교는 그대로 유지합니다.
- Agent API, Viewer 화면, 장비 등록 흐름, 설정 파일과 저장 데이터 형식은 변경하지 않았습니다.
- 장비에 전달되는 읽기 전용 `show` 명령과 Telnet 세션 정책은 변경하지 않았습니다.
- 인증서 변경이 감지된 연결은 계속 차단되며 `AGENT_IDENTITY_CHANGED`로 구분합니다.

## 검증

- 명시적 연결 확인의 반복 재검증
- 정상 Test/Execute 요청의 신원 확인 재사용
- 동시 최초 요청의 단일 신원 확인
- Agent 전송 실패 후 다음 요청의 재검증
- 명시적 재검증 실패 뒤 이전 캐시 사용 금지
- identity 본문 지문 불일치 시 Telnet 요청 차단
- 인증서 변경 상태의 동시성 안전성
- Core, Agent, Viewer 전체 자동화 테스트
- Release 빌드, 패키지 계약, PowerShell 5.1 구문, Git 공백과 GitHub 릴리스 계약 검증

## 알려진 제한

- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어에서 검증하기 전까지는 POC 상태입니다.
- 실제 사내 방화벽, EDR, Windows 서비스 계정과 장시간 운영은 사내 단계에서 확인해야 합니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없으며 사내 EDR 또는 WDAC 승인이 필요할 수 있습니다.
