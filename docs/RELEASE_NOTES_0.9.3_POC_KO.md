# Samsung Switch Watch 0.9.3-poc 릴리스 노트

## 핵심 변경

이번 버전은 Agent가 HTTPS 신원을 처음 만드는 도중 전원 종료나 프로세스 중단이 발생해도
다음 시작에서 안전하게 복구하도록 초기 생성 절차를 보강한 안정성 릴리스입니다.

- 최초 생성 시작을 나타내는 내부 표시 파일을 인증서보다 먼저 기록하고 디스크에 반영합니다.
- 인증서와 메타데이터는 같은 데이터 폴더의 임시 파일에 완전히 기록한 뒤 최종 파일명으로
  이동합니다.
- 인증서와 메타데이터가 모두 완성된 뒤에만 내부 표시 파일을 제거하고 Agent를 시작합니다.
- 생성 도중 중단되어 표시 파일과 일부 파일만 남은 경우에는 해당 생성 파일만 정리하고
  새 신원을 만듭니다.
- 표시 파일 자체의 기록이 중단된 경우에도 다른 신원 파일이 전혀 없을 때만 표시 파일을
  정리하고 최초 생성을 다시 시작합니다.
- 두 Agent 프로세스가 최초 생성을 거의 동시에 시작해도 표시 파일을 얻은 뒤 파일 상태를
  다시 확인하여 먼저 완성된 신원을 삭제하거나 교체하지 않습니다.
- 최종 두 파일이 유효한 상태에서 표시 파일만 남은 경우에는 기존 Agent 신원을 그대로
  사용하므로 Viewer의 신뢰 정보가 바뀌지 않습니다.
- 표시 파일이 없는 기존 신원 손상, 손상된 최종 파일 쌍, 다른 프로세스가 사용 중인 생성
  표시 파일은 자동 교체하지 않고 `TLS_IDENTITY_INVALID`로 안전하게 실패합니다.

## 호환성

- Agent API, Viewer 화면, 장비 명령, 설정 파일과 저장 데이터 형식은 변경하지 않았습니다.
- 최종 Agent 신원 파일명인 `agent-identity.json`과
  `https-certificate.pfx.dpapi`는 그대로 유지합니다.
- 정상 실행이 완료되면 데이터 폴더에는 기존과 같이 최종 신원 파일 두 개만 남습니다.
- 기존 Agent 신원과 Viewer의 저장된 신뢰 정보는 그대로 사용할 수 있습니다.

## 검증

- 생성 표시만 남은 상태, 임시 인증서만 남은 상태, 두 임시 파일이 남은 상태,
  인증서만 최종 경로로 이동된 상태의 재시작 복구 테스트
- 생성 표시 자체가 일부만 기록된 상태의 안전한 복구와 다른 파일 동반 시 실패 테스트
- 두 최종 파일 완료 후 표시 파일 삭제 전에 중단된 상태에서 기존 신원을 보존하는 테스트
- 표시 없는 부분 손상, 손상된 최종 파일 쌍, 표시 없는 임시 파일을 자동 교체하지 않는 테스트
- 생성 표시 파일이 다른 프로세스에 잠긴 경우 파일을 삭제하지 않는 테스트
- 동시 최초 생성과 복구 파일 잠금이 발생해도 기존 신원을 보존하는 테스트
- Core, Agent와 Viewer 전체 자동화 테스트
- Release 빌드, 패키지 계약, PowerShell 5.1 구문, Git 공백과 GitHub 릴리스 계약 검증

## 알려진 제한

- 실제 IES4224GP, IES4028XP, IES4226XP 펌웨어 검증 전에는 POC 상태입니다.
- 전원 차단 순간의 실제 파일 시스템과 사내 EDR 동작은 사내 단계적 검증이 필요합니다.
- Telnet 구간은 평문이므로 격리된 사내 관리망에서만 사용하십시오.
- 공개 `poc` 빌드는 조직 코드 서명이 없으면 사내 EDR 또는 WDAC 승인이 필요할 수 있습니다.
